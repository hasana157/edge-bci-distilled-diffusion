3. PROBLEM DEFINITION
3.1 The Central Challenge
Brain-computer interfaces (BCIs) that help paralyzed or severely disabled individuals require real-time processing: acquire EEG signal → denoise → classify → respond, all within milliseconds. Generative models like Diffusion Models excel at denoising but require hundreds of sequential forward passes, making them impractical for real-time, resource-constrained devices.
3.2 Why Current Approaches Fail on CPU-Only Hardware
Diffusion Models Are Slow
Standard DDPM requires 1000 forward passes for full quality image generation
Even 10-50 steps takes seconds per sample on CPU (vs milliseconds on GPU)
Inference latency: 5-10 seconds per EEG window is UNACCEPTABLE for closed-loop BCIs
Memory overhead: Full model + noise schedules require 2-4 GB RAM
Edge Devices Are Constrained
Jetson Nano, mobile SoCs, and embedded processors have 2-8GB RAM total
Typical BCI systems must run on <500mW power budget (battery constraints)
Real GPU-like compute is unavailable in many research settings
Knowledge Distillation Is The Only Practical Compression Method
Quantization: Loses too much quality for critical medical signals
Pruning: Requires careful architecture redesign (time-consuming)
Distillation: Train a small model to mimic the large one's outputs
  ◦ Preserves signal quality
  ◦ Achieves 10-100x speedup
  ◦ Feasible on CPU (just slower training)
3.3 Why This Project Matters for BCIs
Brings state-of-the-art denoising to devices that currently only support classical filters
Validates that research-grade methods CAN work on constrained hardware
Creates a reusable framework for future edge BCI deployments
Directly applicable to: wheelchair control, communication aids, neurorehabilitation

7.3 Training Pipeline
How models are trained (CPU-optimized):
Phase 1: Diffusion Training
  ◦ 100 epochs, batch size 16 (CPU memory limit)
  ◦ Linear noise schedule (1000 steps), cosine variance adjustment
  ◦ MSE loss (noise prediction), Adam optimizer LR=0.001
  ◦ Checkpoint every 10 epochs, save best model
Phase 2: Distillation Training
  ◦ Trained diffusion model frozen, inference mode
  ◦ Student network trained on synthetic teacher outputs
  ◦ KL divergence (soft targets) + MSE (hard targets)
  ◦ 200 epochs, smaller batch size (32), LR schedule with warm-up
7.4 Inference Pipeline
Real-time deployment scenario:
Input: EEG window (22 channels, 750 samples)
Preprocess: Bandpass 4-40 Hz, z-norm per channel
Denoise: Diffusion or Distilled model (inference only, no backprop)
Feature Extraction: CSP, power spectral density, log-band power
Classify: Motor imagery CNN (softmax 4 classes)
Output: Predicted class, confidence, latency timing
Measure: End-to-end wall-clock time <50ms

8. AI/ML DESIGN SPECIFICATION
8.1 Baseline Approaches (Classical Filtering)
Approach 1: Butterworth Bandpass Filter
Type: IIR recursive filter
Frequency range: 4-40 Hz (motor imagery band)
Order: 4
Pros: Fast (real-time capable), numerically stable, well-understood
Cons: Phase distortion, no adaptive properties
Implementation: scipy.signal.butter + lfilter
Expected SNR improvement: 2-3 dB
Approach 2: Wavelet Denoising (Daubechies-4)
Type: Nonlinear denoising via wavelet thresholding
Decomposition levels: 4-5
Threshold selection: BayesShrink (SURE principle)
Pros: Adaptive, preserves sharp features, handles non-stationary noise
Cons: Slower than Butterworth, threshold selection critical
Implementation: pywt.wavedec + threshold
Expected SNR improvement: 3-4 dB
Approach 3: Wiener Filtering
Type: Optimal linear filter for known signal + noise
Assumes: Gaussian signal + noise, known power spectra
Pros: Mathematically optimal under Gaussian assumptions
Cons: Requires noise PSD estimation, not adaptive
Expected SNR improvement: 3-3.5 dB
8.2 Diffusion Model Design
Architecture: Lightweight UNet-1D
Input: (Batch, 1 channel, 750 samples) at timestep t
Conditioning: Noise level embedding t (sinusoidal encoding)
Layers:
  ◦ Conv1d(1 → 64, kernel 3) + ReLU
  ◦ Residual block × 3 with skip connections
  ◦ Attention at deepest layer (optional, ~50K params)
  ◦ Conv1d transpose to 1 channel output
Total parameters: ~450K (very small by modern standards)
Noise Schedule
Type: Linear schedule from β_start=0.0001 to β_end=0.02
Timesteps T: 1000 (training), 10-50 (inference)
Variance preservation: Cosine schedule adjustment
Rationale: Empirically works well for EEG, simple to implement
Training Objective
Loss: L(θ) = E_t,x [||ε_θ(x_t, t) - ε||²]  (noise prediction)
Optimizer: Adam, LR=0.001, no weight decay
Batch size: 16 (CPU memory constraint)
Epochs: 100
Training time estimate: 12-18 hours on CPU
Inference Process
Reverse diffusion sampling (Euler method):
  x_{t-1} = (1/√α_t) * x_t + (β̃_t / √(1 - ᾱ_t)) * ε_θ(x_t, t)
Steps: Test with 10, 25, 50 (trade-off latency vs quality)
Guidance: None (unconditional), could add classifier-free guidance for future
8.3 Distillation Design
Student Architecture Option 1: Small CNN
Input: (1, 750)
Conv1d(1 → 32, kernel 5) + ReLU + MaxPool1d(2)
Conv1d(32 → 64, kernel 5) + ReLU + MaxPool1d(2)
Flatten → Dense(256) + ReLU + Dropout(0.2)
Dense(750)  [output same shape as input]
Total params: ~85K, inference latency <20ms on CPU
Student Architecture Option 2: Autoencoder
Encoder: Conv1d(1→16→32) with stride=2
Bottleneck: 64-dim latent vector
Decoder: ConvTranspose1d(32→16→1) mirror encoder
Total params: ~45K, ultra-lightweight
Advantage: Unsupervised pretraining possible, interpretable latent
Distillation Training
Temperature-scaled KL divergence:
  L_distill = KL(softmax(teacher/T), softmax(student/T))
Combined loss: 0.75 * L_distill + 0.25 * L_hard
Optimizer: Adam LR=0.0005, schedule decay every 50 epochs
Epochs: 200 (slower convergence, more careful training needed)
Training time: 8-12 hours on CPU
8.4 Justification of Choices
Why DDPM over GANs/VAEs for this project?
GANs: Mode collapse, training instability (bad for CPU-constrained research)
VAEs: Blurry reconstructions due to KL regularization (SNR loss)
DDPM: Stable training, empirically high-quality outputs, proven on medical signals
Cost: Slow inference, but acceptable after distillation
Why Distillation over Quantization?
Quantization: 4-8 bit precision causes significant SNR loss on EEG
Distillation: Can achieve 10-100x speedup with minimal quality loss
CPU-friendly: Distillation only requires training once, inference is native Python
Why No Consistency Models?
Consistency models offer single-step generation (theoretical speedup)
But: Training requires GPU-scale optimization, not feasible on CPU
Distillation is mature, proven, CPU-compatible solution
Can always add consistency models in future work section
